# 개발 진행 기록 — MenuLens

## 프로젝트 타임라인

---

### Day 1 — 아이디어 정의 및 기획

**문제 발견**
해외 여행 중 메뉴판을 보고 무엇을 주문해야 할지 몰라 곤란했던 경험에서 출발. 기존 번역 앱은 텍스트 번역만 해줄 뿐, "이게 맛있는 음식인지", "내 입맛에 맞는지"는 알 수 없었음.

**핵심 인사이트**
> "번역이 아니라 설명이 필요하다. 그리고 나한테 맞는 추천이 필요하다."

**결정 사항**
- Claude Vision API를 핵심 엔진으로 사용
- 취향 프로필 기반 추천 기능 포함
- MVP는 사진 업로드 → 번역+설명+추천 단일 플로우에 집중

---

### Day 1 — 기술 스택 결정

**선택: Next.js + Claude API + Vercel**

선택 이유:
- Next.js: API Routes로 백엔드/프론트 통합 가능, 빠른 개발
- Claude claude-sonnet-4-20250514: Vision + 텍스트 생성 모두 가능한 단일 모델
- Vercel: 별도 서버 설정 없이 즉시 배포 가능

**기각된 대안:**
- GPT-4o Vision → Claude가 한국어 자연어 생성 품질이 더 우수
- 별도 OCR + 번역 API 조합 → 복잡도 증가, 단일 모델로 통합

---

### Day 1 — 프롬프트 설계 및 테스트

**v1 프롬프트 (실패)**
```
이 메뉴판을 번역해줘
```
→ 단순 텍스트 번역만 출력, 설명 없음

**v2 프롬프트 (개선)**
```
메뉴판의 각 항목을 번역하고, 어떤 음식인지 한국인이 이해하기 쉽게 설명해줘
```
→ 설명은 생성되나 포맷이 일정하지 않음

**v3 프롬프트 (채택)**
- JSON 구조 명시
- 취향 프로필 변수 삽입
- 한국 음식 비유 지시문 추가
→ 일관된 구조의 응답 생성 확인

**테스트 케이스:**
- 일본어 이자카야 메뉴 ✅
- 프랑스어 레스토랑 메뉴 ✅
- 손글씨 메뉴판 ⚠️ (인식률 70%, 개선 필요)

### Day 1 — UI/UX 설계

**사용자 플로우**
```
앱 실행
→ 취향 설문 (최초 1회, 30초)
→ 메뉴판 촬영 또는 업로드
→ AI 분석 (로딩 2-3초)
→ 번역+설명 카드 목록
→ 추천 Top 3 하이라이트
→ 즐겨찾기 저장
```

**핵심 UI 원칙**
- 한 손으로 사용 가능한 모바일 우선 설계
- 분석 중 로딩 애니메이션으로 대기 경험 개선
- 음식 설명은 짧고 임팩트 있게 (3줄 이내)

### 회고 및 배운 점

**잘 된 것**
- Claude 단일 모델로 OCR + 번역 + 설명 + 추천을 모두 처리 → 아키텍처 단순화
- JSON 포맷 프롬프트로 프론트엔드 연동 매끄럽게 처리

**어려웠던 것**
- 이미지 품질에 따른 인식률 편차 → 재촬영 가이드로 보완
- 프롬프트 튜닝에 예상보다 많은 시간 소요

**다음에 시도할 것**
- 음식 사진도 함께 보여주는 기능 (Google Places API 연동)
- 가격 비교 및 "이 가격이 적당한가" 현지 시세 안내
